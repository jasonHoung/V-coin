# Coin 經濟模型設計

## 一、基本參數

### 1.1 總量與分配
```
總發行量: 1,000,000,000 VCoin (10億)
初始流通: 0 (全數鎖倉，透過算力挖礦釋放)
```

### 1.2 分配結構
| 類別 | 比例 | 數量 | 解鎖方式 |
|------|------|------|----------|
| GPU 算力獎勵 | 55% | 550,000,000 | 根據算力貢獻逐步釋放 |
| 生態與市場 | 20% | 200,000,000 | 24個月線性解鎖 |
| 團隊 | 15% | 150,000,000 | 36個月線性解鎖，首12個月全鎖 |
| 協議儲備 | 10% | 100,000,000 | 治理投票決定使用 |

## 二、鑄造機制

### 2.1 算力鑄造公式

```
每日鑄造量 = Base × Decay^(天數/365)
Base = 2,000,000 VCoin/日
Decay = 0.85 (每年遞減15%)
```

**年度釋放預估**
- 第1年: ~547,000,000 VCoin
- 第2年: ~88,000,000 VCoin
- 第3年: ~52,000,000 VCoin
- 後續逐年遞減

### 2.2 算力貢獻計算

```
節點獎勵 = (節點算力貢獻 / 全網總算力) × 當日鑄造量 × 信譽係數
```

**算力單位換算**
- 1 CUDA Core-Hour = 1 算力點
- RTX 3090: 10,496 CUDA Cores → 每小時 10,496 算力點
- A100: 6,912 CUDA Cores → 每小時 6,912 算力點（但效能加權 1.5×）

**信譽係數**
```
信譽分數 >= 90: 係數 1.0
信譽分數 80-89: 係數 0.9
信譽分數 70-79: 係數 0.7
信譽分數 < 70: 無獎勵
```

## 三、消耗機制

### 3.1 推論服務定價

| 模型類型 | 每次請求 | 燃燒比例 |
|---------|----------|---------|
| LLM-7B (類似 Llama) | 0.1 VCoin | 10% |
| LLM-13B | 0.2 VCoin | 10% |
| LLM-70B | 1.0 VCoin | 10% |
| Stable Diffusion | 0.5 VCoin | 15% |

**流向**
- 10-15% 燃燒
- 85-90% 支付給執行節點

### 3.2 訓練服務定價

```
訓練費用 = 基礎費用 + GPU-Hours × 單價
基礎費用 = 10 VCoin（任務啟動費）
GPU-Hour 單價 = 2 VCoin/小時（RTX 3090 等級）
```

**範例**
- LoRA 訓練（8 小時，2張 GPU）
  - 費用 = 10 + (8 × 2 × 2) = 42 VCoin
  - 其中 20% 燃燒 = 8.4 VCoin
  - 節點獲得 = 33.6 VCoin

## 四、質押機制

### 4.1 節點質押要求

| GPU 等級 | 最低質押 | 推薦質押 |
|---------|---------|---------|
| 消費級 (RTX 3090) | 1,000 VCoin | 5,000 VCoin |
| 專業級 (A100) | 5,000 VCoin | 20,000 VCoin |
| 資料中心級 | 20,000 VCoin | 100,000 VCoin |

**質押收益**
- 基礎: 無額外收益（僅作為保證金）
- 進階: 質押量 > 推薦值，獎勵係數 +10%

### 4.2 Slashing 規則

| 違規類型 | 懲罰比例 | 條件 |
|---------|---------|------|
| 輕度作弊 | 5% | 單次驗證失敗 |
| 中度作弊 | 15% | 3次驗證失敗 |
| 重度作弊 | 50% | 刻意偽造結果 |
| 長期離線 | 1%/日 | 超過7天未回應 |

**Slashed Coin 流向**
- 50% 燃燒
- 50% 進入協議儲備

## 五、通縮設計

### 5.1 燃燒來源

1. **消耗燃燒**: 每次推論/訓練消耗的 10-20%
2. **懲罰燃燒**: Slashing 的 50%
3. **閒置燃燒**: 長期未使用的帳戶（超過2年）年費 1%

### 5.2 通縮預估

**假設場景**
- 日均推論量: 1,000,000 次
- 平均消耗: 0.2 VCoin/次
- 燃燒率: 10%

```
日燃燒量 = 1,000,000 × 0.2 × 0.1 = 20,000 VCoin/日
年燃燒量 ≈ 7,300,000 VCoin
```

**長期平衡點**
- 當年度燃燒量 = 年度鑄造量時達到通縮平衡
- 預計第 3-5 年達成

## 六、幣價穩定機制

### 6.1 算力定錨

```
1 VCoin 目標錨定價值 = 0.5 USD（參考市場 GPU 租用成本）
實際市價 由供需決定
```

**動態調整**
- 若幣價 > 目標價 50%，增加鑄造量 10%
- 若幣價 < 目標價 50%，減少鑄造量 10%
- 每季度調整一次

### 6.2 流動性管理

**協議儲備使用**
- 50% 用於 DEX 流動性池
- 30% 用於市場穩定基金
- 20% 治理緊急儲備

## 七、與算力的兌換比例

### 7.1 推論算力

```
1 VCoin ≈ 10次 LLM-7B 推論 ≈ 2次 LLM-70B 推論
市場參考: OpenAI GPT-3.5 約 0.002 USD/1K tokens
```

### 7.2 訓練算力

```
1 VCoin ≈ 0.5 GPU-Hour (RTX 3090)
市場參考: AWS p3.2xlarge (V100) 約 3 USD/小時
平台定價約為市場價 40-60%
```

## 八、經濟模型總結

### 8.1 核心原則
1. **實用價值**: 幣價由實際算力成本支撐
2. **通縮設計**: 長期燃燒機制確保稀缺性
3. **激勵相容**: 節點、用戶、平台三方利益一致

### 8.2 風險控制
- 最低質押要求防止女巫攻擊
- Slashing 機制懲罰作弊
- 動態調整避免幣價劇烈波動

### 8.3 成長預期

**保守場景**（前3年）
```
年度推論量: 100M → 500M → 1B
年度訓練量: 10K → 50K → 200K 任務
總鎖倉價值: 10M → 50M → 200M USD
```

**樂觀場景**（5年）
```
成為主要 AI 算力平台之一
日均消耗 > 日均鑄造
進入通縮階段
幣價穩定在算力錨定價附近
```
